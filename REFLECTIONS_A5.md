# Reflection – Assignment 5

One of the main challenges in this assignment was translating system requirements into use cases and test cases. While the functional requirements defined what the system should do, converting them into user interactions required careful thinking about how each stakeholder would interact with the system.

Another challenge was identifying the relationships between use cases, such as inclusion and dependency. For example, capturing grades requires validation, which had to be represented correctly in the diagram.

Developing test cases also required ensuring that each functional requirement could be tested clearly. It was important to define steps and expected results in a way that could verify system behavior accurately.

Additionally, balancing simplicity and completeness was challenging. The system needed to remain simple enough to manage, while still covering all stakeholder needs and system requirements.

Overall, this assignment helped in understanding how system requirements are translated into practical system interactions and validation processes.
