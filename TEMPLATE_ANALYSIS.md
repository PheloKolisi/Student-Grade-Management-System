# Template Analysis – GitHub Projects

## Comparison of GitHub Project Templates

| Template         | Columns/Workflow          | Automation Features                        | Agile Suitability              |
| ---------------- | ------------------------- | ------------------------------------------ | ------------------------------ |
| Basic Kanban     | To Do, In Progress, Done  | No automation                              | Suitable for simple projects   |
| Automated Kanban | To Do, In Progress, Done  | Automatically moves issues based on status | Highly suitable for Agile      |
| Bug Triage       | New, Needs Review, Closed | Auto-labeling and issue tracking           | Suitable for bug tracking only |

---

## Selected Template: Automated Kanban

### Justification

The **Automated Kanban template** was selected because it supports Agile development through built-in automation features.

* It automatically moves tasks between columns based on their status.
* It reduces manual work when managing tasks.
* It aligns with Agile principles such as continuous delivery and workflow visibility.
* It supports linking issues directly to tasks, which improves traceability.

This template is ideal for the **Student Grade Management System** because it simplifies task tracking and allows efficient sprint management.
