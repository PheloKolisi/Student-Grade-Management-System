# User Stories – Student Grade Management System

| Story ID | User Story                                                                                | Acceptance Criteria                | Priority |
| -------- | ----------------------------------------------------------------------------------------- | ---------------------------------- | -------- |
| US-001   | As a lecturer, I want to add student details so that I can store student records.         | Student details saved successfully | High     |
| US-002   | As a lecturer, I want to add subjects so that I can assign grades.                        | Subject saved in system            | High     |
| US-003   | As a lecturer, I want to capture student grades so that academic performance is recorded. | Marks between 0–100 accepted       | High     |
| US-004   | As a lecturer, I want to update grades so that corrections can be made.                   | Updated grades are saved           | High     |
| US-005   | As a lecturer, I want to view student grades so that I can monitor performance.           | Grades displayed correctly         | High     |
| US-006   | As a student, I want to view my grades so that I can track my progress.                   | Student sees correct grades        | Medium   |
| US-007   | As an admin, I want to generate reports so that I can analyze results.                    | Report generated successfully      | Medium   |
| US-008   | As a lecturer, I want to delete student records so that outdated data is removed.         | Record deleted successfully        | Low      |

### Non-Functional User Story

| US-009 | As a system admin, I want data to be secure so that student records are protected. | Data access restricted | High |
